# IMI WORK 철학적 기반: 7가지 루트

> "당신의 모든 비즈니스 결정을 설명하는 철학적 토대"

## 개요

이 폴더는 **이림의 모든 비즈니스 활동**(컨설팅, 교육, 카페 운영, IMI WORK 브랜드)을 설명하는 **7가지 철학적 루트(Philosophical Roots)**를 담고 있습니다.

처음에는 "일을 잘한다는 것"과 "프로세스 이코노미" 2개의 뿌리라고 생각했지만, 깊이 분석한 결과 **이 둘은 더 근본적인 7개 철학에서 파생된 응용 개념**이었습니다.

|------|----------|---------------|

| **[[root-02-의미가치]]** | 효용을 넘어선 가치 창출 | 평균이 아닌 강점, 브랜딩 철학 |

## 루트의 3계층 구조

### Layer 1: 기반 (Foundation)
**감각**과 **의미가치**
- 모든 것의 출발점
- AI 시대 인간의 차별화

### Layer 2: 실행 (Execution)
**과도함**, **프로세스 공유**, **인사이드 아웃**
- 기반을 어떻게 실현할 것인가
- 행동의 원칙

### Layer 3: 지속 (Sustainability)
**현장감**과 **아카이브 사고**
- 어떻게 오래 지속하고 축적할 것인가
- 시간의 힘 활용

## "일을 잘한다는 것"과 "프로세스 이코노미"의 재해석

### 일을 잘한다는 것 =
- **감각** (기술 너머의 통찰)
- \+ **의미가치** (효용→의미)
- \+ **인사이드 아웃** (내면 확신)

### 프로세스 이코노미 =
- **프로세스 공유** (과정 중심)
- \+ **의미가치** (의미 창출)
- \+ **아카이브 사고** (과정 축적)

**결론**: 이 둘은 독립된 루트가 아니라 **7개 루트의 특정 조합**

## IMI WORK 철학 방정식

```
IMI WORK = 감각(15년 경험)
           × 과도함(AI 자동화)
           × 프로세스 공유(OSMU)
           × 현장감(Playing Coach)
           × 아카이브(PKM)
```

## AI 시대의 경쟁력 구조

### AI가 대체 못 하는 것 (당신의 경쟁력)
- **감각**: 맥락과 취향의 이해
- **현장감**: 15년 오프라인 경험
- **인사이드 아웃**: 자신만의 철학

### AI가 증폭시키는 것 (생산성 배수)
- **아카이브 사고**: Claude Code + PKM
- **프로세스 공유**: 자동화된 OSMU
- **과도함**: n8n 워크플로우, 무한 반복 실험

## 각 루트 상세 보기

1. **[[root-01-감각]]** - 기술 너머의 인간적 통찰
   - 핵심: AI 딸깍 시대, 진짜 경쟁력은 인간의 감각
   - 적용: SENSE & AI 브랜드 철학

2. **[[root-02-의미가치]]** - 효용을 넘어선 가치 창출
   - 핵심: 평균점에 돈을 지불하는 사람은 없다
   - 적용: 브랜딩, 컨설팅 철학

3. **[[root-03-과도함]]** - 평범함을 거부하는 실행력
   - 핵심: 사람들이 과도하다고 말하는 것이 그냥 성과다
   - 적용: 6개 서브에이전트, 3개월 집중 프로젝트

4. **[[root-04-프로세스-공유]]** - 과정을 통한 연결과 소속감
   - 핵심: 사람들은 무엇이 아니라 왜에 지갑을 연다
   - 적용: 블로그 과정 공개, GPTers 함께 만들기

5. **[[root-05-인사이드-아웃]]** - 내면 동기 기반 행동
   - 핵심: 외부 정보가 아닌 자신의 확신에서 출발
   - 적용: "나는 이렇게 생각한다" 관철, 트렌드 무시

6. **[[root-06-현장감]]** - 책상이 아닌 실제 현장 이해
   - 핵심: Playing Coach 리더십
   - 적용: 15년 오프라인 경험, 현장 방문 컨설팅

7. **[[root-07-아카이브-사고]]** - 과정을 기록하고 축적
   - 핵심: 시간이 지날수록 가치 증가 (복리 효과)
   - 적용: PKM Vault, n8n 백업, OSMU 시스템

## 실전 활용법

### 1. 의사결정 할 때
**"이 결정이 어떤 루트에 기반하는가?"** 자문

- 직관적 판단 → [[root-01-감각]]
- 차별화 고민 → [[root-02-의미가치]]
- 더 밀어붙일까 고민 → [[root-03-과도함]]
- 공유할까 말까 고민 → [[root-04-프로세스-공유]]
- 남들 따라할까 내 길 갈까 → [[root-05-인사이드-아웃]]
- 분석할까 직접 해볼까 → [[root-06-현장감]]
- 기록할까 말까 → [[root-07-아카이브-사고]]

### 2. 프로젝트 회고할 때
**"이 프로젝트는 어떤 루트를 활용했고, 어떤 루트가 부족했는가?"**

성공 요인 = 활용된 루트
개선 지점 = 부족한 루트

### 3. 타인 설명할 때
**"이 사람은 어떤 루트가 강한가?"**

- 감각형: 디자이너, 브랜더
- 과도함형: 창업자, 혁신가
- 현장감형: 오퍼레이터, 매니저
